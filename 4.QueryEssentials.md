# Query Essentials

## 1. Basic SQL Commands Structure

The fundamental structure of a SQL query involves these key clauses:

| Clause | Purpose |
| :--- | :--- |
| **SELECT** | Specifies **which columns** you want to retrieve. |
| **FROM** | Specifies **which table** you are querying. |
| **WHERE** | Filters **which rows** to retrieve based on conditions. |
| **DISTINCT** | Removes duplicate rows from the result. |
| **ORDER BY** | Sorts the result set in ascending (ASC) or descending (DESC) order. |
| **LIMIT** | Restricts the number of rows returned. |
| **OFFSET** | Skips a specific number of rows. |
| **AS** | Assigns an **alias** (temporary name) to a column or table. |
| `*` | A wildcard that selects **all columns**. |

## 2. SQL Operators

### Arithmetic Operators
Used for mathematical operations on numeric data.
- `+` (Addition)
- `-` (Subtraction)
- `*` (Multiplication)
- `/` (Division)
- `%` (Modulus/Remainder)

### Comparison Operators
Used in the `WHERE` clause to filter data.
- `=` : Equal to
- `!=` or `<>` : Not equal to
- `>` : Greater than
- `<` : Less than
- `>=` : Greater than or equal to
- `<=` : Less than or equal to

### Logical Operators
Used to combine multiple conditions.
- **AND**: Returns true if *both* conditions are true.
- **OR**: Returns true if *at least one* condition is true.
- **NOT**: Reverses the value of a condition.
- **IN**: Checks if a value matches any value in a list.
- **BETWEEN**: Checks if a value is within a range.
- **LIKE**: Searches for a specified pattern in a column.
- **IS NULL** / **IS NOT NULL**: Checks for empty (NULL) values.

## 3. Practical Example: Countries Database

### Setup: Create Database and Table

```sql
-- Create and select the database
CREATE DATABASE IF NOT EXISTS my_db;
USE my_db;

-- Create the table
DROP TABLE IF EXISTS countries;
CREATE TABLE countries (
  id INT PRIMARY KEY AUTO_INCREMENT,
  country VARCHAR(64) NOT NULL,
  population BIGINT NULL,
  area INT NOT NULL,
  region VARCHAR(64) NOT NULL
);

-- Insert sample data
INSERT INTO countries (country, population, area, region) VALUES
('India',     1400000000, 3287000, 'Asia'),
('China',     1412000000, 9597000, 'Asia'),
('Brazil',     214000000, 8516000, 'South America'),
('Australia',   25700000, 7692000, 'Oceania'),
('USA',        331000000, 9834000, 'North America');
```

## 4. Query Examples

### Basic Selection
```sql
-- Select all columns from the table
SELECT * FROM countries;

-- Select specific columns
SELECT country, population FROM countries;
```

### Filtering Data (WHERE Clause)
```sql
-- Exact match
SELECT * FROM countries WHERE region = 'Asia';

-- Not equal to
SELECT * FROM countries WHERE country != 'USA';

-- Numeric comparison (Population less than or equal to 1.2 Billion)
SELECT * FROM countries WHERE population <= 1200000000;
```

### Using Logical Operators
```sql
-- AND: Both conditions must be true
SELECT * FROM countries 
WHERE population > 200000000 AND area > 4000000;

-- OR: At least one condition must be true
SELECT * FROM countries 
WHERE region = 'Asia' OR region = 'Oceania';
```

### Sorting and Limiting (ORDER BY, LIMIT)
```sql
-- Sort by area in descending order and take the top 2
SELECT * FROM countries 
ORDER BY area DESC 
LIMIT 2;
```

### Using DISTINCT and Aliases
```sql
-- Get unique regions and rename the column in output
SELECT DISTINCT region AS my_region FROM countries;
```
